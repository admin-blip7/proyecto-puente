```typescript
// Fire-and-forget WhatsApp notification (REQ-010)
// IMPORTANT: Do NOT await — this must never block the corte flow (REQ-013)
const waBranchId = userProfile?.branchId ?? null;
if (waBranchId) {
  fetch('/api/whatsapp/corte', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      sessionId: closedSession.id,
      branchId: waBranchId,
    }),
  }).then(async (res) => {
    if (!res.ok) return; // HTTP error from our own route — already logged server-side
    const json = await res.json().catch(() => ({}));
    if (json?.ok === false && json?.error && json.error !== 'no_number') {
      toast({
        title: "WhatsApp",
        description: "No se pudo enviar la notificación de corte.",
        variant: "destructive",
      });
    }
  }).catch(() => {
    toast({
      title: "WhatsApp",
      description: "No se pudo enviar la notificación de corte.",
      variant: "destructive",
    });
  });
}
```

Rules:
- Do NOT add `await` before `fetch(...)` — fire-and-forget is required
- `no_number` errors are silently swallowed (expected when WhatsApp not configured)
- The outer `try/catch` of handleCloseDrawer already exists — do not duplicate it
- Make only this minimal insertion. Do not modify any other logic in the function.
- If `userProfile?.branchId` is undefined or null, skip the call entirely (branch not set = no notification)
